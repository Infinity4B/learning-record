# 背景

![[SmolVLA overview.png]]

VLA模型处于发展阶段，目前很多有影响力的VLA模型只贡献权重，没有分享训练细节和关键方法组件。主张开发负担得起的高效模型，通过透明、可复现的开源模型和训练方法来促进机器人研究的发展。[[OpenVLA]]和RT-2-X展示了开放VLA系统的可行性，但是模型参数量大，需要昂贵的机器人平台。

提出了SmolVLA，一个采用紧凑但是能力强的开源VLA模型，附有可复现的高效训练和推理代码。贡献如下：

- 轻量级架构：提出SmolVLA，一个紧凑高效的视觉语言agent，专为消费级 GPU 训练和 CPU 部署优化。关键设计选择包括：（1）跳过 VLM 的层；（2）使用最少的视觉token；（3）利用小型预训练VLM；（4）将自注意力层与轻量级交叉注意力层交织。
- 社区驱动的数据集预训练：SmolVLA 在不到 30000 个 episode 的社区贡献的数据集上端到端训练，展示了强性能，同时使用的数据量比之前模型小了一个数量级。
- 异步推理：提出了一个优化的异步推理栈，将动作执行与观察处理和动作预测解耦，减少延迟并实现快速、资源高效的推理。

# 模型

SmolVLA 是一个轻量级的 VLA，由一个紧凑的预训练 VLM 和一个用流匹配训练的动作专家组成。给定多个图像和一个描述任务的语言指令，模型输出一系列的动作。首先，模型在社区收集的数据集上用模仿学习预训练，然后在现实世界和模拟环境中进行评估。预训练数据旨在涵盖多种任务和行为，使模型能够学习在不同环境中都能转移的通用物理技能。在推理阶段，引入了一个异步执行栈，将动作执行与感知和预测解耦，从而实现更快、响应性更强的控制。

## 模型架构

SmolVLA包含一个预训练VLM和一个动作专家。VLM处理状态输入，将特征传给动作专家。动作专家输出动作，改变状态。

### VLM

选择SmolVLM-2，利用SIgLIP编码视觉特征，SmolLM2作为语言解码器。视觉特征同样利用线性层投影到语言空间。语言解码器层的结果传到动作专家中。

### 状态、动作和特征投影

利用了三个线性投影层：

- 将状态映射到VLM维度
- 将动作映射到动作专家维度
- 将VLM特征对齐到动作专家维度

### 视觉token压缩

高分辨率图像会增加推理成本。SmolVLM-2采用图像分割训练，图像会有多个切片，以及全局图像。为了实现更快的推理时间，不使用分割。仅使用全局图像，并结合像素打乱操作，将视觉token限制为每帧64个。

### 通过跳过层实现更快推理

在VLM中跳过计算来实现更快推理。已有工作发现最优的特征不一定是从VLM的最后一层得到。因此让动作专家最多只能接触到指定 $N$ 层。实际实验中设置 $N$ 为总层数的一半，达到了速度和性能的平衡。

### 流匹配动作专家

动作专家 $\text{v}_\theta$ 目标是从VLM特征中预测动作块 $\mathbf{A}_t=(a_t,\dots,a_{t+n})$ ，同样利用的是Transformer架构。和之前VLA设计不同，交织交叉注意力和自注意力层，利用条件流匹配Transformer作为动作专家。训练目标为：

$$
\mathcal{L}^{\tau}(\theta) = \mathbb{E}_{p(\mathbf{A}_t \mid \mathbf{o}_t), q(\mathbf{A}_t^\tau \mid \mathbf{A}_t)} \left[ \left\| \mathbf{v}(\mathbf{A}_t^{\tau}, \mathbf{o}_t) - \mathbf{u}(\mathbf{A}_t^{\tau} \mid \mathbf{A}_t) \right\|^2 \right]
$$

其中 $\mathbf{o}_t$ 代表了观察 $o_t$ 从第 $N$ 个VLM层提取出的特征。 $\mathbf{A}_t^\tau=\tau\mathbf{A}_t+(1-\tau)\epsilon,\epsilon\sim\mathcal{N}(0,\mathbf{I})$。动作专家训练从VLM特征和带噪动作 $\mathbf{A}_t^\tau$ 中预测向量场 $\mathbf{u}(\mathbf{A}_t^\tau\mid\mathbf{A}_t)=\epsilon-\mathbf{A}_t$ 。和 [[Pi-0|π0]] 一样，从Beta分布采样 $\tau$ 。为了提高推理效率，将动作专家的隐藏维度变为原来的0.75倍。

### 交织交叉注意力和因果自注意力层

动作专家 $\text{v}_\theta$ 基于 VLM 特征生成动作块，VLM 和动作专家的交互通过注意力机制实现。与先前完全依赖自注意力（如 $\pi_0$ ）或交叉注意力（如 GR00t-n1）的工作不同，采用交织方法，每个块包含交叉注意力或自注意力层。这种设计选择也与标准 VLM 架构不同，每个解码器块通常包含自注意力和 交叉注意力层。在动作专家的前向传递中，动作与 VLM 特征的交互通过注意力进行，将 token 投影到qkv中。交叉注意力层交叉关注 VLM 的键和值，而自注意力层允许 $\text{v}_\theta$ 中的动作 token 彼此关注。为自注意力层使用因果注意力掩码，确保每个动作 token 只能关注块内的过去 token，防止未来动作依赖。实际研究发现，交织交叉注意力和自注意力层达到了更高的成功率和更快的推理时间。此外，还发现自注意力有助于更流畅的动作块 $\mathbf{A}$ ，在实机机器人评估中这一点尤为明显。

## 社区采集的预训练数据

机器人数据的整合和扩展很复杂，因为不同数据集有不同设定，并且依赖人类遥操来收集。不同机器人本体、传感器、动作模块、控制频率和数据格式形成了数据异质性，给数据集整合带来困难。

与遵循标准化协议的学术数据集不同，社区数据集自然涵盖了多种本体、控制方案、摄像头视角和任务。社区数据集通过嘈杂的演示、异质环境和多样化的对象交互，反映了现实世界的复杂性，为预训练数据提供了有价值的参考。从Hugging Face中选取了481个社区数据集，并根据本体类型、episode数量、整体数据质量和帧覆盖进行筛选。共22.9K个episode，10.6M帧。

## 异步推理

目前的很多策略都使用了动作块的概念 $\pi(o_t)=\mathbf{A}_t, \mathbf{A}_t=(a_t,a_{t+1},\dots,a_{t+n})$ 。传统的机器人会在新观察 $o_{t+n}$ 传到策略预测下一动作块之前执行完整个动作块。这导致了观察之间开环的推理。ACT和Diffusion Policy采用了完全不同的策略，机器人控制器把动作预测 $\mathbf{A}_t\gets\pi(o_t)$ 和块消耗 $a_t\gets \text{POPFRONT}(\mathbf{A})$ 交织在一起，在每个时间步 $t$ 预测新的动作块并且在重叠部分聚合预测的动作块。并且在重叠部分聚合预测的动作块。这样的方法需要持续推理，在资源有限场景中难以实现。

用完动作块再预测的方式被称为同步推理，在计算动作块时空闲会带来延迟，并且实时反应能力弱。为了解决这一开环问题，提出了异步推理的方法。其中 $\text{ROBOCLIENT}$ 发送观察 $o_t$ 到 $\text{ROBOSERVER}$ ，推理完成接收动作块 $\mathbf{A}_t$ 。通过在控制循环仍在使用先前可用的队列时触发动作块预测，并与新传入队列合并，从而避免执行延迟。反过来，异步推理通过提高处理分块预测的频率，从而在动作预测与动作执行之间建立更紧密的关系。将动作预测与动作执行解耦，还可直接在远程策略服务器上分配更多计算资源，通过网络将动作发送给机器人客户端，这在资源受限的场景（如低功耗机器人）中可能非常有效。

![[SmolVLA model1.png]]

在算法执行层面， $\text{ROBOCLIENT}$ 一直执行已有的队列，直到剩余队列数量满足阈值 $|\mathbf{A}_t|/n<g$ 。条件触发时，捕捉新观察发送到 $\text{POLICYSERVER}$ 。 为了避免冗余的服务器调用，两个距离小于阈值的观察会被判定为重复观察，并被丢弃。当可用队列为空时，无论如何都会处理观察。

对于推理而言，主要延时来源于推理时间 $\mathbb{E}[\ell_S]$ ，两者之间发送延迟可以忽略不计。令 $\Delta t$ 为环境的控制循环时间，为了在运行时没有空闲时间，需要 $g \ge \frac{\mathbb{E}[\ell_S]/\Delta t}{n}$ 。

![SmolVLA model2.png]]

上图展示了动作块大小对于不同 $g$ 值随时间变化的图：

- 序列限制（ $g=0$ ）：执行所有动作块后再传入新的观察，可以看到有部分时间机器人没有执行行动，带来 $\mathbb{E}[\ell_S]$ 的空闲时间。
- 异步推理（ $g=0.7$ ）：大约执行 30% 的序列才执行新动作块的推理，避免了误差出现，也避免了密集计算。
- 计算密集限制 （ $g=1$ ）：和[[ACT]]和[[Diffusion Policy]]一样，每一时间步都计算，带来高昂的计算成本。

上图(B)红色箭头则是展示了一个时间步，在这一步观测相似性机制被绕过，由于队列已经空了，所以即使是相似的观察也会被处理。
